**프로젝트명**: HI We


**기간**: Nov–Dec 2023 (~1 month)


**역할**: 프론트엔드 전반 담당


## 1. **프로젝트 소개**


HI We는 후원이 필요한 **동물과 단체**를 후원자와 연결하는 커뮤니티 플랫폼입니다.


단순한 후원 기능에 그치지 않고, 커뮤니티·마이페이지 등 다양한 기능을 제공하여 **후원자가 지속적으로 참여하고 교류할 수 있는 환경**을 목표로 했습니다.


## 2. **기술 스택**

- **Frontend**: React, Next.js, TypeScript
- **상태 관리**: Recoil
- **데이터 통신**: GraphQL
- **기타**: 카카오 주소 API, Infinite Scroll 구현, 반응형 UI

## 3. **주요 기능 및 시연**


### 랜딩 및 회원 시스템

- 반응형 랜딩 페이지 구현 (PC/모바일 대응)
- 로그인/회원가입 기능 → GraphQL API 연동
- 회원 정보 저장 및 관리

## 4. **개발 과정에서의 도전과 해결**

- **전역 상태 관리**: Recoil을 활용하여 후원/회원 데이터 전역 관리 → Context API 대비 가볍고 직관적인 구조 경험
- **데이터 통신**: GraphQL 기반 API 요청 및 캐싱 처리, REST API 대비 구조화된 요청 설계 학습
- **주소 처리**: 카카오 주소 API를 연동하여 실제 서비스에서 활용 가능한 위치 검색 기능 구현
- **UX 개선**: Infinite Scroll을 적용하여 게시글/후원 목록 로딩 시 사용자 경험 개선

## 5. **협업 경험**

- 백엔드와 GraphQL API 협업 → 명세 기반 개발, 요청-응답 구조 설계 경험
- 팀 내 Git Flow 활용 → 브랜치 전략, 코드 리뷰 경험
- 실제 서비스와 유사한 개발 사이클 경험

## 6. **프로젝트에서 배운 점**

- Next.js 환경에서의 **서버사이드 렌더링(SSR) 이해**
- GraphQL의 장점(불필요한 데이터 요청 방지, 명확한 스키마 기반 통신) 직접 체감
- **상태 관리(Recoil)와 데이터 흐름 관리**에 대한 이해도 상승
- 사용자 경험(UX) 중심으로 기능을 설계하고 구현하는 역량 강화

## 7. **정리**


HI We 프로젝트는 단순한 토이 프로젝트를 넘어서,


**실제 서비스에 가까운 기능 구현과 협업 경험**을 통해 제 역량을 증명할 수 있는 좋은 사례가 되었습니다.

- 기술적으로는 **Next.js, TypeScript, Recoil, GraphQL**을 깊게 다뤄본 경험이 되었고,
- 협업 측면에서는 **API 연동과 Git Flow 기반 개발**을 경험하면서 실무 감각을 익혔습니다.

앞으로는 이 경험을 바탕으로 **더 확장성 있는 구조 설계**와 **실제 배포 경험**까지 이어가고 싶습니다.
