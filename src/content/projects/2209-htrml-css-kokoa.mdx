## 1. **프로젝트 소개**


이 프로젝트는 **카카오톡(KakaoTalk) UI를 클론 코딩**하며 HTML과 CSS의 기초를 연습하기 위해 진행했습니다.


순수 HTML과 CSS만을 사용하여 UI를 구현하면서 **웹 레이아웃과 스타일링 기초**를 다지는 것이 목표였습니다.

## 2. **학습 포인트**

- **HTML 구조 설계**
    - 시멘틱 태그(header, nav, section, footer 등)를 활용
    - 접근성을 고려한 기본 마크업
- **CSS 기초 문법 정리**
    - 박스 모델 이해
    - Flexbox와 Grid를 통한 레이아웃 구성
    - position, z-index로 화면 요소 제어

💻 _[코드 스니펫 예시 — Flexbox 정렬]_


```css
.chat-list {
  display: flex;
  flex-direction: column;
  gap: 12px;
}

.chat-item {
  display: flex;
  align-items: center;
  justify-content: space-between;
}
```


## 3. **성과 및 배운 점**

- **HTML & CSS 기반의 레이아웃 구현 능력**이 향상됨
- Flexbox와 Grid를 활용한 모바일 UI 구성 능력 습득
- 반복 학습으로 코드 작성 속도와 일관성이 개선됨
- 작은 UI의 **디테일(여백, 폰트, 색상)**이 사용자 경험에 미치는 영향 체감

## 4. **다음 학습 계획**

- JavaScript DOM 조작을 활용한 동적인 UI 구현 연습
- 반응형 웹(Responsive Web)과 미디어 쿼리 학습
- React/Next.js 같은 프레임워크에서 동일 UI를 확장해보기

💻 _[코드 스니펫 예시 — 미디어 쿼리 준비]_


```css
@media (max-width: 768px) {
  body {
    font-size: 14px;
  }
  .chat-list {
    gap: 8px;
  }
}
```


## 5. **정리**


이 프로젝트는 포트폴리오용 완성작이 아니라, **학습 기록**입니다.


작은 화면이라도 직접 만들어보며 **기초 문법과 원리를 몸에 익히는 과정**이었고,


이를 통해 이후 프레임워크 학습으로 넘어갈 수 있는 **탄탄한 기반**을 다졌습니다.

- 앞으로는 JavaScript와 React를 연계해 “정적 UI”에서 “동적인 웹앱”으로 발전시켜 나갈 계획입니다.
